# ระบบที่ปรึกษานิสิต (Student Advisor System) - ภาพรวมและแผนการดำเนินงาน

## ภาพรวมระบบ

### วัตถุประสงค์
ระบบจัดการการขอคำปรึกษาจากอาจารย์ที่ปรึกษาสำหรับนิสิต โดยนิสิตสามารถส่งคำขอได้พร้อมกัน 5 คน และเมื่อมีอาจารย์ 1 คนรับเป็นที่ปรึกษา คำขออื่นๆ จะถูกยกเลิกอัตโนมัติ

### กลุ่มผู้ใช้งาน
1. **นิสิต (Student)** - ขอคำปรึกษาจากอาจารย์
2. **อาจารย์ที่ปรึกษา (Advisor)** - รับ/ปฏิเสธคำขอ และจัดการนิสิตที่ปรึกษา
3. **ผู้ดูแลระบบ (Administrator)** - จัดการข้อมูลพื้นฐานและผู้ใช้งาน

### ฟีเจอร์หลัก
- การลงทะเบียนและเข้าสู่ระบบ (Roleless Login)
- การค้นหาและเลือกอาจารย์ที่ปรึกษา
- การส่งคำขอและอัปโหลดเอกสาร
- การอนุมัติ/ปฏิเสธคำขอ
- การจัดการข้อมูลพื้นฐาน (Master Data)
- การจัดการผู้ใช้งาน

## สถาปัตยกรรมระบบ

### Database (MySQL)
- **ตารางหลัก**: student, advisor, administrator, faculty, department, program, expertise, academic_term, request
- **ตารางความสัมพันธ์**: advisor_expertise (many-to-many)
- **ตารางเสริม**: academic_rank, academic_degree, student_advisor (optional)
- **Views**: auth_accounts, advisor_capacity_view
- **Security**: SHA2-256 password hashing

### Backend (PHP)
- **API Structure**: RESTful API with JSON responses
- **Authentication**: Session-based with roleless login
- **Security**: CORS, Rate Limiting, Prepared Statements, Transaction Management
- **File Handling**: PDF upload with validation

### Frontend (Next.js 15 + React 19)
- **Framework**: Next.js 15 with App Router
- **Styling**: Tailwind CSS 4
- **State Management**: React Hooks
- **API Integration**: Fetch wrapper with error handling

## รายละเอียดการพัฒนา

   - Header with navigation

2. **Form Components**
   - Input fields with validation

3. **Data Display**

   - Cards for advisor profiles
   - Status badges
   - Progress indicators

4. **Interactive Elements**
   - Modals for confirmations
   - Toast notifications
   - Loading spinners
   - Error messages

### API Endpoints ที่ต้องเชื่อมต่อ
1. **Authentication**
   - POST /api/auth/login
   - POST /api/auth/logout

2. **Student Management**
   - POST /api/students (register)
   - GET /api/students/:id
   - PUT /api/students/:id

3. **Advisor Management**
   - POST /api/advisors (register)
   - GET /api/advisors (list with filters)
   - GET /api/advisors/:id
   - PUT /api/advisors/:id

4. **Request Management**
   - POST /api/requests (submit)
   - GET /api/requests (list)
   - GET /api/requests/:id
   - PUT /api/requests/:id/status
   - POST /api/requests/upload-file

5. **Master Data**
   - GET /api/helpers/faculties
   - GET /api/helpers/departments
   - GET /api/helpers/programs
   - GET /api/helpers/expertises
   - GET /api/helpers/academic_terms

6. **Admin Management** (ต้องสร้างใหม่)
   - GET /api/admin/users
   - GET /api/admin/users/:id
   - PUT /api/admin/users/:id
   - DELETE /api/admin/users/:id
   - POST /api/admin/users/:id/reset-password

### Database Tables ที่ต้องเพิ่ม
1. **User Management**
   - user_status (active, suspended, deleted)
   - user_audit_log (track changes)

2. **System Configuration**
   - system_settings (configurable parameters)
   - notification_templates
