## วัตถุประสงค์
เอกสารนี้ใช้สำหรับอัดวิดีโอพรีเซนโปรเจกต์ระบบที่ปรึกษานิสิต โดยสรุปข้อมูลเดโมที่ควรใช้ ลำดับการพรีเซน และบทพูดภาษาไทยแบบพร้อมอ่าน

## ชื่อโปรเจกต์
Project Advisor System
ระบบยื่นคำขออาจารย์ที่ปรึกษาโครงงานสำหรับนิสิต

## จุดเด่นที่ควรพูดให้ชัด
- ระบบรองรับผู้ใช้ 3 บทบาท: นิสิต, อาจารย์ที่ปรึกษา, ผู้ดูแลระบบ
- นิสิตสามารถส่งคำขอหาอาจารย์ได้หลายคน พร้อมติดตามสถานะได้
- เมื่อมีอาจารย์ตอบรับ ระบบมี business logic จัดการคำขออื่นที่เกี่ยวข้องให้อัตโนมัติ
- อาจารย์สามารถจัดการคำขอ ดูความจุการรับนิสิต และแก้ไขข้อมูลส่วนตัวได้
- ผู้ดูแลระบบสามารถจัดการข้อมูลพื้นฐานและผู้ใช้งานได้จากหน้า admin
- มีระบบลืมรหัสผ่านและตั้งรหัสผ่านใหม่

## สคริปต์พรีเซนแบบพร้อมอ่าน

### 1. เปิดคลิป
สวัสดีครับ ในวิดีโอนี้ผมจะนำเสนอโปรเจกต์ Project Advisor System หรือระบบยื่นคำขออาจารย์ที่ปรึกษาโครงงานสำหรับนิสิต

ระบบนี้ถูกออกแบบมาเพื่อช่วยให้กระบวนการเลือกอาจารย์ที่ปรึกษาเป็นระบบมากขึ้น ทั้งในฝั่งนิสิต อาจารย์ และผู้ดูแลระบบ

### 2. อธิบายปัญหาและแนวคิด
ปัญหาที่โปรเจกต์นี้ต้องการแก้ คือในหลายกรณีการเลือกอาจารย์ที่ปรึกษายังทำแบบกระจัดกระจาย เช่น ติดต่อกันเองผ่านแชตหรือเอกสาร ทำให้ติดตามสถานะได้ยาก และผู้ดูแลระบบตรวจสอบภาพรวมได้ลำบาก

ดังนั้นระบบนี้จึงถูกพัฒนาขึ้นมาเพื่อให้การยื่นคำขอ การอนุมัติ การติดตามสถานะ และการจัดการข้อมูลทั้งหมดอยู่ในระบบเดียว

### 3. อธิบายบทบาทผู้ใช้
ระบบนี้มีผู้ใช้ 3 กลุ่มหลัก

กลุ่มแรกคือนิสิต สามารถค้นหาอาจารย์ ดูรายละเอียด และส่งคำขอพร้อมเอกสารได้

กลุ่มที่สองคืออาจารย์ที่ปรึกษา สามารถดูคำขอที่เข้ามา อนุมัติหรือปฏิเสธ และบริหารจำนวนที่รับได้

กลุ่มที่สามคือผู้ดูแลระบบ สามารถจัดการผู้ใช้งานและข้อมูลพื้นฐานของระบบ เช่น คณะ ภาควิชา และข้อมูลประกอบอื่น ๆ

### 4. เดโมหน้า Login
ตอนนี้ผมอยู่ที่หน้าเข้าสู่ระบบ ซึ่งรองรับการ login แบบใช้ email และ password และเมื่อเข้าสู่ระบบสำเร็จ ระบบจะพาไปยัง dashboard ตามบทบาทของผู้ใช้โดยอัตโนมัติ

### 5. เดโมฝั่งนิสิต
ต่อไปผมจะเข้าสู่ระบบในบทบาทของนิสิต

หลังจากเข้าสู่ระบบแล้ว จะเห็นหน้า dashboard ของนิสิต ซึ่งแสดงภาพรวมของคำขอและข้อมูลที่เกี่ยวข้อง

จากนั้นนิสิตสามารถเข้าไปดูรายชื่ออาจารย์ที่ปรึกษา ซึ่งในส่วนนี้สามารถใช้เพื่อค้นหาอาจารย์ที่เหมาะสมตามข้อมูลในระบบ

เมื่อเลือกอาจารย์ได้แล้ว นิสิตสามารถยื่นคำขอ พร้อมแนบเอกสารประกอบ เช่น proposal หรือไฟล์ที่ระบบกำหนด

อีกส่วนสำคัญคือระบบสามารถติดตามสถานะคำขอได้ ทำให้นิสิตรู้ได้ทันทีว่าคำขออยู่ในขั้นตอนไหน เช่น รอพิจารณา อนุมัติ หรือเสร็จสิ้น

จุดเด่นของฝั่งนิสิตคือช่วยลดความสับสนในการติดตามคำขอ และทำให้ทุกอย่างอยู่ในรูปแบบดิจิทัลที่ตรวจสอบได้

### 6. เดโมฝั่งอาจารย์
ต่อไปผมจะออกจากระบบและเข้าสู่ระบบในบทบาทของอาจารย์ที่ปรึกษา

เมื่อเข้ามาในฝั่งอาจารย์ จะเห็น dashboard ที่สรุปข้อมูลคำขอและภาระการรับนิสิต

ในส่วนของการจัดการคำขอ อาจารย์สามารถเปิดดูรายละเอียดของคำขอแต่ละรายการ และตัดสินใจอนุมัติหรือปฏิเสธได้

นอกจากนี้อาจารย์ยังสามารถจัดการข้อมูลโปรไฟล์ของตัวเองได้ เช่น ข้อมูลส่วนตัว ความเชี่ยวชาญ หรือจำนวนความจุในการรับนิสิต

จุดเด่นในฝั่งนี้คือช่วยให้อาจารย์บริหารคำขอได้ง่ายขึ้น และเห็นภาพรวมของนิสิตที่ตนเองดูแลได้ชัดเจน

### 7. เดโมฝั่งผู้ดูแลระบบ
สุดท้ายผมจะเข้าสู่ระบบในบทบาทของผู้ดูแลระบบ

ในหน้า admin dashboard ผู้ดูแลระบบจะเห็นภาพรวมของระบบทั้งหมด ทั้งจำนวนผู้ใช้และข้อมูลที่เกี่ยวข้องกับคำขอ

ผู้ดูแลระบบสามารถไปยังหน้าจัดการผู้ใช้งาน เพื่อดูข้อมูลของนิสิต อาจารย์ และผู้ดูแลระบบ รวมถึงแก้ไขข้อมูลบางส่วนได้

นอกจากนี้ยังมีส่วนจัดการข้อมูลพื้นฐาน หรือ master data เช่น คณะ ภาควิชา ความเชี่ยวชาญ และข้อมูลประกอบอื่น ๆ ซึ่งทำให้ระบบสามารถดูแลข้อมูลหลักได้จากส่วนกลาง

จุดเด่นของฝั่งผู้ดูแลระบบคือช่วยลดภาระในการดูแลข้อมูลแบบ manual และทำให้ระบบพร้อมใช้งานในระดับองค์กรหรือระดับคณะได้มากขึ้น

### 8. อธิบายเทคโนโลยี
สำหรับเทคโนโลยีที่ใช้ ในส่วน frontend พัฒนาด้วย Next.js และ React ส่วน backend ใช้ PHP และฐานข้อมูล MySQL

ระบบมีการจัดการ session สำหรับการยืนยันตัวตน มีการแยกสิทธิ์ตามบทบาทของผู้ใช้ และมีการจัดการ API สำหรับเชื่อมต่อข้อมูลระหว่าง frontend กับ backend

### 9. อธิบาย business logic สำคัญ
อีกจุดหนึ่งที่สำคัญคือ business logic ของระบบ เช่น การจัดการคำขอของนิสิต การติดตามสถานะคำขอ และการรีเซ็ตรหัสผ่านในกรณีลืมรหัสผ่าน ซึ่งช่วยให้ระบบมีความสมบูรณ์และพร้อมใช้งานจริงมากขึ้น

### 10. ปิดการพรีเซน
สรุปแล้ว Project Advisor System เป็นระบบที่ช่วยให้กระบวนการเลือกอาจารย์ที่ปรึกษาเป็นระบบมากขึ้น ตรวจสอบได้ง่ายขึ้น และลดงานซ้ำซ้อนของทั้งนิสิต อาจารย์ และผู้ดูแลระบบ
